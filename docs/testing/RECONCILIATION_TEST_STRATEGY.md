# Reconciliation Test Strategy

## Layers

- Contract tests validate strict serialization, versions, weights, stage order,
  blocking requirements, and workflow compatibility.
- Unit tests cover normalization audit, Decimal/date boundaries, comparison
  categories, schema structure mapping, composite integrity, stage consumption,
  candidate limits, fuzzy ties/cancellation, and weighted contributions.
- Persistence tests prove append-only superseding decisions and audited,
  project-scoped decision memory.
- Vertical tests import immutable CSV sources, run a background workflow, inspect
  progress/checkpoints, reopen Excel, validate manifests, and compare source hashes.
- Performance tests execute synthetic exact and candidate-blocked fuzzy cases.
- Vitest and Playwright cover the functional Reconciliation Studio journey.

## Fixture coverage

`tests/fixtures/reconciliation` includes reordered/renamed structures, added,
removed, modified, duplicate, null and composite keys, normalization, numeric and
date boundaries, floating precision, fuzzy confidence/ties/blocking/limits,
weighted scoring, integrity violations, review history, formula-injection text,
and synthetic scale inputs generated by the benchmark harness.

No test removes assertions or bypasses budgets to obtain a green result.
